Segmentation of Cellular Structures

We choose the segmentation of certain cell organelles as a relevant Image to Image task. Segmentation is an important tool in EM image analysis as it contributes to a better visualisation of certain organelles and complex cell structures, which facilitates the interpretation of EM data. Segmentation allows for detailed analysis of organelle morphology, spatial relationships and distribution within cells. This is crucial for understanding intracellular organisation and its relationship to cell function. Due to the small available dataset sizes we deploy data augmentation methods, make use of pretrained weights and train an ensemble model, which has been shown to provide better generalizability even when trained on smaller dataset sizes [1].

[1] Shaga Devan, Kavitha, et al. "Weighted average ensemble-based semantic segmentation in biological electron microscopy images." Histochemistry and Cell Biology 158.5 (2022): 447-462.

DEEP-EM TOOLBOX: Workflow

Task

We categorize this segmentation as an Image to Image task, as we require an image as input which will then be translated into a semantic mask, also represented as an image. In the case of this notebook, we follow the work of [1] for defining the model. As a task-specific architecture, we use an encoder-decoder network. More specifically, we make use of the U-Net [2] architecture, which was especially designed for biomedical image segmentation. Our goal is to develop a so-called "ensemble" model. An ensemble model is a set of models that are used to make multiple predictions for the same input data. The predictions are then combined by a weighted average between all trained models. The weights are computed by the performance of the model on the validation set. For the ensemble, we use multiple CNN backbone architectures and include them in our task-specific encoder-decoder architecture. Similar to our previous example notebook, we are using CNN architecture, as it is specially developed for images and performs well on small dataset sizes, compared to ViT backbones. This allows us to get a more robust prediction, even though the training dataset is comparably small.

Data

Similar to the first example use case, we make use of pre-existing real data from [3] for data acquisition. Still, we encourage the research community to test the provided script on their own data. This requires gathering and annotating the data. We again propose the usage of the CVAT [4] annotation tool. By doing so, the annotated data can simply be uploaded and plugged into the already provided pipeline. For details, we refer the reader to our project page and corresponding notebook. As annotation type we are working with pixel-level annotations of segmentation masks, which are color-coded images grouping similar cellular structures, namely nucleus, cytoplasm, and background.

For data preprocessing, we split the dataset into training, validation, and test sets using 60%-20%-20% splits. Lastly, we deploy different augmentations to the input images to increase the variance in the rather small datasets. The applied augmentations include vertical flipping, which flips the images along the vertical axis, and random 90-degree rotations that rotate the images by 90 degrees in any direction. We also apply horizontal flipping to mirror the images along the horizontal axis, and transposing to swap the image's width and height. Additionally, grid distortion is used to introduce slight warps and distortions across the image grid. These transformations are applied on-the-fly with a probability of one, which means that each training sample is slightly altered every time before it is presented to the model. This ensures a large variance in the training data, thus improving the model's ability to generalize from the data.

We reformat our data from single-channel grayscale images to 3-channel RGB images so that we are able to use pretrained model weights where the model was trained on natural images. We do so because the number of pretrained models for EM is very limited, but we require multiple pretrained backbones to initialize the ensemble model. To address the difference between the pretraining data and our EM data, we normalize the EM pixel intensities similarly to z-score normalization but match the mean and standard deviation of the pretraining data. This helps reduce the gap between natural images used during pretraining and the EM images.

Model

As mentioned earlier, we are falling back on pretrained models of natural images for the initialization of our model. Despite the domain gap between natural images and EM, pretrained weights from ImageNet [5] can deliver helpful prior information to the backbone.

We use the standard output as well as Weights & Biases for logging metrics for every model of the ensemble structure. Metrics include training and validation loss as well as IoU on the validation set, which is a metric specific for assessing the performance of segmentation models.

Finally, during evaluation we accumulate the predictions of the models and pick the most likely prediction, forming the ensemble model. We assess the performance quantitatively by computing the IoU for each model separately on the test set and finally combine their predictions to compute the IoU for the ensemble model.

We add qualitative evaluation on a subset of the test data by visualizing the model's input, output, and corresponding ground truth. Again, we assess this for each model separately as well as on the combined ensemble model.

Use Your Own Data

Here we explain how you need to preprocess and annotate your data to apply the model for your use case. We require that all data for training, testing and validation is annotated and collected and then uploaded to the pytorch lightning studio.

Data Structuring

We require you to split the data in a test and train set. Our provided notebook will automatically pick a validation split (20%) from the provided training data.

You should provide one folder containing the training images and one folder containing the corresponding training masks, which were exported from the CVAT tool as described below.

Similarly, you should provide one folder containing the test images and one folder containing the corresponding test masks, which were exported from the CVAT tool as described below.

We recomment to use at least 20% of your dataset for testing.

Within the provided notebook we descibe where you need to adapt the code in order to plug in your data.

Data Annotation

For data annotation we recomment using the CVAT tool. It is free to use but requires a user account. A quick user guide can be found here. Multiple instruction videos can also be found on the LearnOpenCV youtube channel (for example this). For annotating the location labels within this use case we implement the use of point annotations. We export the data using the Segmentation mask 1.1 format.

When starting CVAT, you first need to create a new task. You can give it a name, add annotation types and upload your data.

Next, click on the Add label button. Name it based on the classes you want to annotate. In our case these are "naked", "budding", "enveloped". As annotation type choose Points. You should also pick a color, as this will simplify the annotation process. For adding new labels click Continue. Once you added all nessecary labels click Cancel.

Now you can upload the data you wish to annotate. Finally, click Submit & Open to continue with the annotation of the uploaded data.

This will open following view. Click on the job (in this view the Job #1150327) to start the annotation job.

To then annotate your data, select the Draw new mask tool. Select the Label you wish to annotate from the dropdown menu. Then click Shape to annotate individual virus capsids with the label class.

When you finished annotating a single mask, click on the tick in the top left corner. You can use the arrows on the top middle to navigate through all of your data and to see your annotation progress.

Once you are done annotating data, click on the Menu and select Export job dataset.

During export select the Segmentation mask 1.1 format and give the folder a name. It will prepare the dataset for download.

In the horizontal menu bar at the top go to Requests. It will show a request Export Annotations. On the right of this request click on the three dots to download your annotated data. And you are done.
