Within this image-to-value(s) tasks, we are developing a regression model to quantify HCMV capsids and their maturation stages during secondary envelopment in TEM images. Secondary envelopment of HCMV is the process by which viral capsids acquire a final envelope by budding into cytoplasmic vesicles, a step essential for the production of infectious progeny [1]. During this process, different maturation stages - naked, budding or enveloped capsids - can be observed in TEM images [2]. Quantifying these stages and comparing wild-type viruses with mutants that have defects in secondary envelopment can provide valuable insights into the proteins involved and their role in this critical process [2,3,4]. In our notebook we aim to detect the number of naked, budding and enveloped virus particles within an input image. We use pre-trained model weights to avoid over-fitting on the training data provided. We also focus on additional explanatory techniques (GradCAM [5]) to make the model more trustworthy, reliable and easier to detect incorrect predictions.

DEEP-EM TOOLBOX: Workflow

Task

We categorize the task of this use case into Image to Value(s) as the input will be an EM image, and the output will be three values defining the number of naked, budding, and enveloped HCMV capsids in the input image. Next, we define the model architecture: for selecting the backbone, we take into account that we are working with image data, which opens the possibility of using CNN or ViT architectures. Given the limited amount of data, we choose to use CNN models. Specifically, we select ResNet50 [1] because it allows us to utilize pretrained EM weights from CEM500k [2]. Regarding the task-specific architecture, we follow the standard of using the backbone as a feature encoder and defining a task-specific head with an output dimensionality of three, matching the number of values to predict the secondary envelopment stages within the TEM images.

Data

For data acquisition, we make use of pre-existing real data from [3] that has already been annotated. However, we encourage the research community to gather their own datasets to use our example use case. The annotation type of the data are instance annotations marking capsid locations with classes \(C=\{\text{"naked"}, \text{"budding"}, \text{"enveloped"}\}\). For your own data, we recommend using the CVAT annotation tool [4], as this will allow you to directly plug in your data to run the provided script. For more details, we refer the reader to our project page and the corresponding notebook. Based on these annotations, we can derive the number of naked, budding, and enveloped virus capsids.

For data preprocessing, we split the dataset into training, validation, and test sets, using 20% for testing, 16% for validation, and 64% for training. We choose a relatively large test split, as dataset sizes in EM are usually small. By doing so, we can ensure that the model is sufficiently evaluated to account for potential overfitting to the training data. For resizing we crop random patches of 224×224 pixels from the full-resolution images, to match the input size of our pretrained backbone during training. We balance image patches containing virus capsids with patches that do not contain virus capsids through informed sampling of the patch location. This allows us to account for potential bias in the dataset, as the occurrence of virus capsids is not uniformly distributed. On the validation and test set we work in a sliding window fashion. We apply image normalization by z-score normalization, where we use the mean and standard deviation from the pretrained dataset. This is done to match the distribution of the pretrained dataset and better reuse the learned features from pretraining.

To further increase variance in the dataset, we apply standard augmentation techniques like flipping and rotating the input images.

Model

As mentioned before, we initialize the backbone model with EM-specific pretrained weights from CEM500k [2]. We also log important metrics using Weights & Biases, such as training loss, validation loss, and validation mean average error. We visualize a subset of the validation set during training with model input, output, prediction, and ground truth. This allows us to quantitatively and qualitatively assess the training development of the model. We specifically deploy explainability using GradCAM [5], which helps us to sidestep the black-box nature of the model.

Lastly, we evaluate the overall performance of the model on the test set. We do this in a quantitative and qualitative fashion and again use GradCAM as a method of explaining the model's predictions. This allows us to visualize possible biases in the trained model.

Data Annotation

For data annotation we recomment using the CVAT tool. It is free to use but requires a user account. A quick user guide can be found here. Multiple instruction videos can also be found on the LearnOpenCV youtube channel (for example this). For annotating the location labels within this use case we implement the use of point annotations. We export the data using the CVAT for images 1.1 format.

When starting CVAT, you first need to create a new task. You can give it a name, add annotation types and upload your data.

Next, click on the Add label button. Name it based on the classes you want to annotate. In our case these are "naked", "budding", "enveloped". As annotation type choose Points. You should also pick a color, as this will simplify the annotation process. For adding new labels click Continue. Once you added all nessecary labels click Cancel.

Now you can upload the data you wish to annotate. Finally, click Submit & Open to continue with the annotation of the uploaded data.

This will open following view. Click on the job (in this view the Job #1150022) to start the annotation job.

To then annotate your data, select the Draw new points tool. Select the Label you wish to annotate from the dropdown menue. Then click Shape to annotate individual virus capsids with the label class. (Track will allow you to place annotations over multiple frames, which is helpful when annotating videos, tomograms or similar). You can use the arrows on the top middle to navigate through all of your data and to see your annotation progress.

Once you are done annotating data, click on the Menu and select Export job dataset.

During export select the CVAT for Images 1.1 format and give the folder a name. It will prepare the dataset for download.

In the horizontal menu bar at the top go to Requests. It will show a request Export Annotations. On the right of this request click on the three dots to download your annotated data. And you are done.
